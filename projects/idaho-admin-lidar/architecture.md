# architecture

## Container layout

Three OCI images, all built with podman, all run with
`--device nvidia.com/gpu=all`. None of them talk to each other; they
share data via host bind mounts.

```
inflection-mask3d           inflection-imgseg              inflection-splat
─────────────────           ─────────────────              ────────────────
nvidia/cuda:11.7.1-devel    pytorch:2.4.1-cuda12.1        nerfstudio base
torch 1.13.1+cu117          torch 2.4.1+cu121             splatfacto
MinkowskiEngine 02fc608     transformers 4.57.1           (retired path)
detectron2 (pinned)         huggingface_hub 0.34.4
pointnet2 _ext.so baked     OneFormer + SAM 2 + DINO
TORCH_CUDA_ARCH_LIST=       (downloads weights to a
  7.5;8.0;8.6                mounted HF cache volume)
(4090 = sm_89, runs via
 SASS forward compat)
```

The mask3d image is the painful one — it pins to versions Mask3D's
authors validated against, plus an sm_89 workaround so kernel launches
on a 4090 don't fall back to PTX JIT. Bumping any of those versions is
almost always a multi-day debug session, which is why the Dockerfile
exists at all instead of an environment.yml.

## Where each piece runs

| Step | Container | Hardware | Wall time |
|------|-----------|----------|-----------|
| 01–02 | host PDAL | CPU, 32 GB RAM | ~3 min |
| 03 | host Python | CPU | <30 s |
| 04 | inflection-mask3d | RTX 4090 | ~17 min |
| 05–08 | host Python | CPU | ~2.5 min total |
| 13 | inflection-imgseg | RTX 4090 | ~30 s (95 panos) |
| 14 | inflection-imgseg | RTX 4090 | ~30 s (95 panos) |
| 15 | host Python (torch) | RTX 4090 | ~10 s |
| 16 | host Python (torch) | RTX 4090 | ~10 s |
| 17 | host Python + PDAL | CPU | ~30 s |
| upload | aws-cli | (network) | ~30 s |

Total wall time end-to-end: ~25 minutes for the full 240×320 m site.

## Why containerized at all

Mask3D's dependency stack is unrecoverable on a modern host without a
container. Torch 1.13.1+cu117, MinkowskiEngine pinned to a specific
commit, detectron2 from a specific git SHA, pyyaml 5.4.1 built
no-isolation against an old Cython, hydra-core 1.0.5 with a specific
omegaconf and antlr4 — every one of those is a single command on a
clean image and a brick wall on a system Python. The image isn't
about deployability; it's about reproducibility on *one* machine.

The image-fusion stack (OneFormer + Grounded-SAM-2) is its own image
because transformers 4.57.1 (which is where Sam2Processor lives)
won't co-install with Mask3D's pinned torch version.
